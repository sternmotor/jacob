* 64 bit ist 32 bit und 32 bit ist 64 bit
    https://social.technet.microsoft.com/Forums/scriptcenter/en-US/6cf0f6bc-164e-4fa1-8612-476cf4de90e7/powershell-script-sharepoint-scheduled-task-fails?forum=ITCG

    32 bit - %windir%\System32\WindowsPowerShell\V1.0 
    64 bit - %windir%\Syswow64\WindowsPowerShell\V1.0
    SHOULD BE
    64 bit - %windir%\System32\WindowsPowerShell\V1.0 
    32 bit - %windir%\Syswow64\WindowsPowerShell\V1.0
    The default on 64 is SYSTEM32.  I know - it sounds backwards but that is how MS got it to work.
    If we open a shortcut to the 32 bit PowerShell (x86) this is what it shows:
    %SystemRoot%\syswow64\WindowsPowerShell\v1.0\powershell.exe
    Notice that the 32 bit version is in SYSWOW64.  SYSWOW64 stands for System Windows On Window 64.  Again a bad choice of names.  The OS default is called WOW or Windows on Windows.  That now runs as Windows On Windows 64 meaning that we are running traditional 32 bit Windows in NEW Windows 64.  At least that was the half-baked Microsoft explanation.
    Getting this backwards can and will drive you nuts.
